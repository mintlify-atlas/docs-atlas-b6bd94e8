---
title: Environment Variables
description: Required API keys and configuration variables for Drift
---

Drift requires several API keys and configuration variables to function. These should be stored in a `.env` file at the root of your monorepo.

## Required Variables

### Nessie API (Banking Data)

<CodeGroup>
```bash .env
NESSIE_API_KEY=your_nessie_api_key_here
NESSIE_BASE_URL=http://api.nessieisreal.com
```
</CodeGroup>

<ParamField path="NESSIE_API_KEY" type="string" required>
  Capital One Nessie API key for accessing demo banking data. Get your key at [nessieisreal.com](http://nessieisreal.com).
</ParamField>

<ParamField path="NESSIE_BASE_URL" type="string" default="http://api.nessieisreal.com">
  Base URL for the Nessie API. Defaults to the public Nessie endpoint.
</ParamField>

### Google Gemini (AI Goal Parsing)

<CodeGroup>
```bash .env
GEMINI_API_KEY=your_gemini_api_key_here
```
</CodeGroup>

<ParamField path="GEMINI_API_KEY" type="string" required>
  Google Gemini API key for natural language goal parsing, narrative generation, and conversational AI. Get your key from [Google AI Studio](https://aistudio.google.com/app/apikey).
  
  **Used for:**
  - Parsing user goals from plain English ("Save $50k for a house in 3 years")
  - Generating personalized financial narratives
  - Powering voice conversation features
  - Creating actionable recommendations
</ParamField>

<Warning>
  Without a Gemini API key, the app falls back to mock goal parsing with limited pattern matching.
</Warning>

### ElevenLabs (Voice Narration)

<CodeGroup>
```bash .env
ELEVENLABS_API_KEY=your_elevenlabs_api_key_here
ELEVENLABS_VOICE_ID=josh
```
</CodeGroup>

<ParamField path="ELEVENLABS_API_KEY" type="string">
  ElevenLabs API key for text-to-speech voice narration and speech-to-text transcription. Get your key at [elevenlabs.io](https://elevenlabs.io).
  
  **Optional** - The app works without voice features if this is not set.
</ParamField>

<ParamField path="ELEVENLABS_VOICE_ID" type="string" default="josh">
  Voice ID for narration. Available options:
  - `josh` - Friendly, energetic (default)
  - `adam` - Deep, authoritative
  - `rachel` - Warm, professional
  - `bella` - Soft, reassuring
  - `antoni` - Confident, punchy
  - `domi` - Strong, bold
</ParamField>

### Plaid (Real Bank Connectivity)

<CodeGroup>
```bash .env
PLAID_CLIENT_ID=your_plaid_client_id
PLAID_SECRET=your_plaid_secret
PLAID_ENV=sandbox
```
</CodeGroup>

<ParamField path="PLAID_CLIENT_ID" type="string">
  Plaid client ID for connecting real bank accounts. Get credentials at [plaid.com/developers](https://plaid.com/developers).
</ParamField>

<ParamField path="PLAID_SECRET" type="string">
  Plaid secret key corresponding to your client ID.
</ParamField>

<ParamField path="PLAID_ENV" type="string" default="sandbox">
  Plaid environment. Options:
  - `sandbox` - Testing with fake data
  - `development` - Testing with real credentials
  - `production` - Live data
</ParamField>

<Info>
  Plaid is optional. The app can use Nessie demo data if Plaid is not configured.
</Info>

## Optional Variables

### OpenAI (Alternative to Gemini)

<CodeGroup>
```bash .env
OPENAI_API_KEY=your_openai_api_key_here
```
</CodeGroup>

<ParamField path="OPENAI_API_KEY" type="string">
  OpenAI API key (used by the Python simulation layer for enhanced analysis). Get your key at [platform.openai.com](https://platform.openai.com).
</ParamField>

### Server Configuration

<CodeGroup>
```bash .env
PORT=3001
NEXT_PUBLIC_API_URL=http://localhost:3001
```
</CodeGroup>

<ParamField path="PORT" type="number" default="3001">
  Port for the Express API server.
</ParamField>

<ParamField path="NEXT_PUBLIC_API_URL" type="string" default="http://localhost:3001">
  API base URL accessible from the Next.js frontend. Must be prefixed with `NEXT_PUBLIC_` to be available in the browser.
</ParamField>

## Complete .env Template

<CodeGroup>
```bash .env
# Required - Nessie Banking API
NESSIE_API_KEY=your_nessie_api_key_here
NESSIE_BASE_URL=http://api.nessieisreal.com

# Required - Google Gemini AI
GEMINI_API_KEY=your_gemini_api_key_here

# Optional - ElevenLabs Voice
ELEVENLABS_API_KEY=your_elevenlabs_api_key_here

# Optional - Plaid Bank Connectivity
PLAID_CLIENT_ID=your_plaid_client_id
PLAID_SECRET=your_plaid_secret
PLAID_ENV=sandbox

# Optional - OpenAI
OPENAI_API_KEY=your_openai_api_key_here

# Server Configuration
PORT=3001
NEXT_PUBLIC_API_URL=http://localhost:3001
```
</CodeGroup>

## Security Best Practices

<Warning>
  **Never commit your `.env` file to version control!** 
  
  The `.env` file should be listed in `.gitignore`.
</Warning>

<Steps>
  <Step title="Use .env.example for templates">
    Create a `.env.example` file with placeholder values to commit to your repository:
    
    ```bash
    NESSIE_API_KEY=your_key_here
    GEMINI_API_KEY=your_key_here
    ```
  </Step>
  
  <Step title="Rotate keys regularly">
    Rotate API keys periodically, especially if they may have been exposed.
  </Step>
  
  <Step title="Use different keys per environment">
    Use separate API keys for development, staging, and production environments.
  </Step>
  
  <Step title="Set appropriate rate limits">
    Configure rate limits in your API provider dashboards to prevent abuse.
  </Step>
</Steps>

## Verifying Configuration

The API server logs warnings for missing required keys on startup:

```bash
Warning: NESSIE_API_KEY not set in environment variables
```

You can verify your configuration by checking the health endpoint:

<CodeGroup>
```bash cURL
curl http://localhost:3001/health
```

```json Response
{
  "status": "ok",
  "timestamp": "2026-03-02T23:04:15.123Z"
}
```
</CodeGroup>
