## Overview

The Recommendations endpoint uses AI to analyze Monte Carlo simulation results and generate specific, actionable financial recommendations. It considers the user's financial profile, goal, and simulation outcomes to provide personalized advice.

## Endpoint

```
POST /llm/recommendations
```

## Request Body

<ParamField path="simulationResults" type="object" required>
  Results from a Monte Carlo simulation run.
  
  <expandable title="properties">
    <ParamField path="successProbability" type="number" required>
      Probability of achieving the goal (0-1)
    </ParamField>
    
    <ParamField path="medianOutcome" type="number" required>
      Median amount saved across all simulations
    </ParamField>
    
    <ParamField path="percentiles" type="object" required>
      Outcome percentiles
      <expandable title="properties">
        <ParamField path="p10" type="number">
          10th percentile (worst likely outcome)
        </ParamField>
        <ParamField path="p25" type="number">
          25th percentile
        </ParamField>
        <ParamField path="p50" type="number">
          50th percentile (median)
        </ParamField>
        <ParamField path="p75" type="number">
          75th percentile
        </ParamField>
        <ParamField path="p90" type="number">
          90th percentile (best likely outcome)
        </ParamField>
      </expandable>
    </ParamField>
    
    <ParamField path="mean" type="number">
      Mean outcome across simulations
    </ParamField>
    
    <ParamField path="std" type="number">
      Standard deviation of outcomes
    </ParamField>
    
    <ParamField path="worstCase" type="number">
      Absolute worst case scenario
    </ParamField>
    
    <ParamField path="bestCase" type="number">
      Absolute best case scenario
    </ParamField>
  </expandable>
</ParamField>

<ParamField path="financialProfile" type="object" required>
  User's current financial situation.
  
  <expandable title="properties">
    <ParamField path="monthlyIncome" type="number" required>
      Monthly income in USD
    </ParamField>
    
    <ParamField path="monthlySpending" type="number" required>
      Monthly spending in USD
    </ParamField>
    
    <ParamField path="liquidAssets" type="number" required>
      Current liquid assets (cash, savings, checking)
    </ParamField>
    
    <ParamField path="creditDebt" type="number" required>
      Total credit card debt
    </ParamField>
    
    <ParamField path="loanDebt" type="number" required>
      Total loan debt
    </ParamField>
    
    <ParamField path="monthlyLoanPayments" type="number" required>
      Monthly loan payment obligations
    </ParamField>
    
    <ParamField path="spendingByCategory" type="object">
      Annual spending breakdown by category (e.g., Food & Dining: 6000, Shopping: 4800)
    </ParamField>
    
    <ParamField path="spendingVolatility" type="number">
      Month-to-month spending volatility (0-1)
    </ParamField>
  </expandable>
</ParamField>

<ParamField path="goal" type="object" required>
  The user's financial goal.
  
  <expandable title="properties">
    <ParamField path="targetAmount" type="number" required>
      Target savings amount in USD
    </ParamField>
    
    <ParamField path="timelineMonths" type="number" required>
      Timeline in months
    </ParamField>
    
    <ParamField path="goalType" type="string" required>
      Type of goal (retirement, major_purchase, emergency_fund, etc.)
    </ParamField>
  </expandable>
</ParamField>

## Response

<ResponseField name="recommendations" type="string[]">
  Array of 3-4 specific, actionable recommendations. Each recommendation:
  - References specific numbers from the user's financial data
  - Provides actionable advice (not vague suggestions)
  - Explains the expected impact on success probability
</ResponseField>

## Implementation Notes

- The endpoint uses Google's Gemini 2.0 Flash model for generating recommendations
- If the Gemini API is unavailable, it falls back to rule-based recommendations
- Recommendations are tailored based on:
  - Success probability (low vs. high)
  - Savings rate (spending vs. income ratio)
  - Risk factors (debt levels, spending volatility)
  - Specific spending categories from transaction data
- Each recommendation includes specific dollar amounts and expected impact
- The AI prioritizes actionable advice over generic suggestions
