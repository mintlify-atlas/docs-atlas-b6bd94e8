---
title: 'Nessie API'
description: 'Access Capital One Nessie banking data including accounts, customers, purchases, deposits, bills, and loans'
---

The Nessie API integration provides access to Capital One's banking sandbox data. All endpoints are prefixed with `/api/nessie`.

## Authentication

Nessie API requires an API key configured in the `NESSIE_API_KEY` environment variable. The API key is automatically appended to all requests.

## Accounts

### Get Accounts by Customer

<CodeGroup>
```bash Request
curl -X GET "http://localhost:3001/api/nessie/accounts?customerId=697541cf95150878eafea4ff"
```

```json Response
[
  {
    "_id": "697541cf95150878eafea500",
    "type": "Checking",
    "nickname": "Main Checking",
    "rewards": 0,
    "balance": 5420.50,
    "account_number": "1234567890",
    "customer_id": "697541cf95150878eafea4ff"
  },
  {
    "_id": "697541cf95150878eafea501",
    "type": "Savings",
    "nickname": "Emergency Fund",
    "rewards": 0,
    "balance": 12500.00,
    "account_number": "0987654321",
    "customer_id": "697541cf95150878eafea4ff"
  }
]
```
</CodeGroup>

<ParamField query="customerId" type="string" required>
  The Nessie customer ID to retrieve accounts for
</ParamField>

<ResponseField name="_id" type="string">
  Unique account identifier
</ResponseField>

<ResponseField name="type" type="string">
  Account type: `Checking`, `Savings`, or `Credit Card`
</ResponseField>

<ResponseField name="nickname" type="string">
  User-friendly account name
</ResponseField>

<ResponseField name="rewards" type="number">
  Rewards points balance
</ResponseField>

<ResponseField name="balance" type="number">
  Current account balance in dollars
</ResponseField>

<ResponseField name="account_number" type="string">
  Account number
</ResponseField>

<ResponseField name="customer_id" type="string">
  Associated customer ID
</ResponseField>

## Customers

### Validate Customer

<CodeGroup>
```bash Request
curl -X POST "http://localhost:3001/api/nessie/validate-customer" \
  -H "Content-Type: application/json" \
  -d '{"customerId": "697541cf95150878eafea4ff"}'
```

```json Response
{
  "valid": true,
  "customer": {
    "_id": "697541cf95150878eafea4ff",
    "first_name": "Alex",
    "last_name": "Morgan",
    "address": {
      "street_number": "123",
      "street_name": "Main St",
      "city": "New York",
      "state": "NY",
      "zip": "10001"
    }
  }
}
```
</CodeGroup>

<ParamField body="customerId" type="string" required>
  Customer ID to validate
</ParamField>

<ResponseField name="valid" type="boolean">
  Whether the customer ID is valid
</ResponseField>

<ResponseField name="customer" type="object">
  Customer details if valid
</ResponseField>

<ResponseField name="customer._id" type="string">
  Customer identifier
</ResponseField>

<ResponseField name="customer.first_name" type="string">
  Customer first name
</ResponseField>

<ResponseField name="customer.last_name" type="string">
  Customer last name
</ResponseField>

<ResponseField name="customer.address" type="object">
  Customer address details
</ResponseField>

## Error Handling

All endpoints return standard HTTP status codes:

- `200 OK` - Request successful
- `400 Bad Request` - Missing or invalid parameters
- `404 Not Found` - Resource not found
- `500 Internal Server Error` - Server error

<CodeGroup>
```json 400 Error
{
  "error": "customerId query parameter is required"
}
```

```json 404 Error
{
  "valid": false,
  "error": "Customer not found"
}
```

```json 500 Error
{
  "error": "Failed to fetch accounts"
}
```
</CodeGroup>
